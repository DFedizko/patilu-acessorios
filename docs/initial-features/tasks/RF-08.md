## 1. Objetivo (o que este RF entrega)

Renderizar a **etiqueta/cartela de código de barras** de uma faixa como **SVG Code128**, pronta para impressão. Tecnicamente: a porta `IBarcodeRenderer`, a implementação `BwipBarcodeRenderer` (bwip-js), o caso de uso `RenderTierLabelUseCase` e a rota `GET /api/tiers/:id/label` que devolve `image/svg+xml`.

## 2. Contexto (por quê)

O código de barras é por **faixa**, não por unidade física: imprime-se uma cartela por faixa (ex.: "Caneta R$1") que fica na bancada, e bipá-la equivale a apertar **+1** daquela faixa (F6 / D12). Este RF entrega o lado "gerar a imagem visível e imprimível" do código único que o RF-07 já criou como dado. A página de etiqueta é Server Component; só o botão de imprimir é client (a impressão em si — `window.print()` + `@page` — é tratada no front, RF-09).

## 3. Início — pré-condições (o que já precisa existir)

- **RF-07 `DONE`:** faixas com **código de barras único** persistido; `TierRepository`/`FindTierByBarcodeUseCase` prontos; rotas `/api/tiers*` em uso; erro `TIER_NOT_FOUND` já no dicionário PT.
- Como conferir: `POST /api/categories/:id/tiers` cria faixa com `barcode` preenchido.

## 4. Escopo — entregáveis (faça exatamente isto)

- `bun add bwip-js` (caso ainda não instalado pelo RF-07).
- `src/server/application/gateway/IBarcodeRenderer.ts` — porta (External Gateway, camada de aplicação): contrato que renderiza um código em SVG.
- `src/server/infrastructure/gateway/BwipBarcodeRenderer.ts` — implementação com **bwip-js**: `toSVG({ bcid: "code128", text, includetext: true })` no servidor.
- `src/server/application/use-case/RenderTierLabelUseCase.ts` + `.../contracts/IRenderTierLabelUseCase.ts` — `{ id }` → SVG string. Busca a faixa (404 `TIER_NOT_FOUND` se não existe), pega o `barcode` e pede o SVG ao renderer.

- `src/server/di/container.ts` + `symbols.ts` — bindings de `IBarcodeRenderer`/`BwipBarcodeRenderer` + `RenderTierLabelUseCase`.

## 8. Trecho da TECH_SPEC que fala deste RF

Código de barras — render (§11):

> - **Render da etiqueta (imagem):** `IBarcodeRenderer` → `BwipBarcodeRenderer` usa **bwip-js** `toSVG({ bcid: "code128", text, includetext: true })` no servidor. Endpoint `GET /api/tiers/:id/label` devolve `image/svg+xml`. A página de etiqueta é Server Component; só o botão de imprimir é client.
> - **Impressão:** **MVP** = `window.print()` + CSS `@media print` com `@page { size: 50mm 30mm; margin: 0 }`. **Depois (QZ Tray)** = impressão silenciosa ZPL/ESC-POS via agente desktop (config de impressora moraria no modal de Configurações). Ver risco §17.
