## 1. Objetivo (o que este RF entrega)

**Negócio:** estabelece a base persistente e de regras de domínio para que o sistema deixe de viver de dados de mentira em memória — todas as tabelas (categorias, faixas, pedidos, empacotamentos, itens, ads por dia, config) passam a existir num banco de verdade.

**Técnico:** entrega o `prisma/schema.prisma` completo (§4) + migração, os Value Objects (`Money`/`Barcode`/`Period`), a taxonomia de erros (`DomainError` + subclasses + `toHttpResponse`), o container de DI do back (Inversify com decorators) e a infraestrutura de banco de teste com helpers/builders.

## 2. Contexto (por quê)

A loja vende kits ao vivo no TikTok sem saber custo nem margem; o produto resolve isso registrando **a faixa de custo de cada item** e calculando a margem ao empacotar. Hoje **não existe back-end** — sem persistência, sem API de domínio, sem auth (PRD §9). Antes de qualquer caso de uso, é preciso a fundação: schema do banco, dinheiro como Value Object (`Money` em centavos), o esqueleto de erros e o DI. Este RF não entrega nenhum endpoint — entrega o chão sobre o qual todos os outros RFs constroem (TECH_SPEC §16.1).

## 7. Trecho da PRD que fala deste RF

**Modelo de domínio (PRD §3)** — as "coisas" que o sistema precisa lembrar:

> | Conceito | O que é | Dados que o sistema lembra |
> | --- | --- | --- |
> | **Categoria** | Agrupamento livre de faixas, só para organizar a tela. | nome; data de criação |
> | **Faixa de custo (tier)** | Rótulo que representa um **preço de compra** (ex.: "Caneta R$1"). | nome; custo (R$); **código de barras gerado pelo sistema**; categoria a que pertence |
> | **Pedido (Order)** | Uma venda vinda do **TikTok**. Não é criado à mão. | id do pedido no TikTok; @cliente; valor da venda; frete; data/hora do pedido; **status de envio**; **status de empacotamento** (interno) |
> | **Empacotamento** | O detalhamento de custo atribuído a um pedido. Um pedido → um empacotamento. | detalhamento dos itens; itens avulsos; custo total; **quem empacotou**; data/hora |
> | **Item do empacotamento** | Quantos itens de uma faixa entraram, com o **custo da faixa congelado**. | faixa de origem (referência); quantidade; **custo unitário no momento** (snapshot); nome da faixa/categoria no momento |
> | **Item avulso** | Item pontual digitado na hora (nome + custo). | nome; custo (R$) |
> | **Custo fixo por pedido** | Custo fixo do negócio por caixa, hoje **R$3**. Configurável. | valor atual configurado (R$) |
> | **Operador (usuário)** | Pessoa logada que empacota. Autenticação via **Clerk**. | identidade do usuário (vinda do Clerk) |

**Pendências para a tech spec (PRD §8)** que esta fundação materializa:

> - Modelagem do banco (categorias, faixas, pedidos, empacotamentos, itens, avulsos, Ads/fallback, custo fixo) e estratégia de _snapshot_.
> - **Custo fixo por pedido:** decidir se o valor **congela** no momento do empacotamento ou se é sempre o **valor atual** aplicado ao período. (Este PRD assume "valor atual aplicado ao período" até decisão contrária.)

**Modelo de dados (§4)** — `prisma/schema.prisma` (provider `postgresql`, client em `src/generated/prisma`, monetário como `Int` centavos):

```prisma
model Category {
    id        String   @id @default(cuid())
    name      String
    createdAt DateTime @default(now())
    updatedAt DateTime @updatedAt
    tiers     Tier[]

    @@map("categories")
}

model Tier {
    id         String    @id @default(cuid())
    name       String
    costCents  Int
    barcode    String    @unique
    categoryId String?                       // null = "Sem categoria"
    category   Category? @relation(fields: [categoryId], references: [id], onDelete: SetNull)
    createdAt  DateTime  @default(now())
    updatedAt  DateTime  @updatedAt

    @@index([categoryId])
    @@map("tiers")
}

enum ShipmentStatus {
    PENDING      // não enviado (AWAITING_SHIPMENT/AWAITING_COLLECTION/UNPAID no TikTok)
    SHIPPED      // IN_TRANSIT/DELIVERED/COMPLETED
    CANCELLED
}

enum PackingStatus {
    NOT_PACKED
    PACKED
}

model Order {
    id             String         @id @default(cuid())
    tiktokOrderId  String         @unique
    orderNumber    String
    recipientName  String?
    saleCents      Int
    shippingCents  Int            @default(0)
    orderedAt      DateTime
    shipmentStatus ShipmentStatus @default(PENDING)
    packingStatus  PackingStatus  @default(NOT_PACKED)  // campo interno nosso
    packing        Packing?
    createdAt      DateTime       @default(now())
    updatedAt      DateTime       @updatedAt

    @@index([orderedAt])
    @@index([shipmentStatus, packingStatus])
    @@map("orders")
}

model Packing {
    id         String        @id @default(cuid())
    orderId    String        @unique
    order      Order         @relation(fields: [orderId], references: [id], onDelete: Cascade)
    operatorId String                                    // Clerk userId
    packedAt   DateTime      @default(now())
    items      PackingItem[]
    looseItems LooseItem[]
    updatedAt  DateTime      @updatedAt

    @@map("packings")
}

// Snapshot congelado da faixa no momento do empacotamento (PRD RN-2.8 / RN-3.5)
model PackingItem {
    id           String  @id @default(cuid())
    packingId    String
    packing      Packing @relation(fields: [packingId], references: [id], onDelete: Cascade)
    tierId       String?                                 // referência fraca (faixa pode ser excluída depois)
    tierName     String                                  // congelado
    categoryName String                                  // congelado (para "custo por categoria")
    unitCostCents Int                                    // congelado
    quantity     Int

    @@index([packingId])
    @@map("packing_items")
}

model LooseItem {
    id        String  @id @default(cuid())
    packingId String
    packing   Packing @relation(fields: [packingId], references: [id], onDelete: Cascade)
    name      String
    costCents Int

    @@index([packingId])
    @@map("loose_items")
}

enum AdSpendSource {
    TIKTOK
    MANUAL
}

// Gasto com anúncios por dia (PRD F5). Período = soma dos dias. (TS16)
model AdSpendDay {
    day        DateTime      @id @db.Date            // YYYY-MM-DD (00:00 SP)
    amountCents Int
    source     AdSpendSource
    updatedAt  DateTime      @updatedAt

    @@map("ad_spend_days")
}

// Config singleton (custo fixo por pedido etc.). Sempre 1 linha (id = "singleton"). (TS17)
model AppConfig {
    id                 String   @id @default("singleton")
    fixedCostPerOrderCents Int  @default(300)         // R$3,00
    updatedAt          DateTime @updatedAt

    @@map("app_config")
}
```

> **"Sem categoria"** não é uma linha: é `Tier.categoryId = null`. Ao excluir uma categoria, `onDelete: SetNull` joga as faixas para `null` automaticamente (PRD RN-1.1).

**Value Objects (§5.1):**

```ts
export class Money {
    private constructor(private readonly cents: number) {}
    static fromCents(cents: number): Money { /* valida inteiro */ }
    static fromReais(reais: number): Money { /* arredonda p/ centavos */ }
    static zero(): Money { return Money.fromCents(0); }
    add(other: Money): Money;
    subtract(other: Money): Money;
    multiplyByQuantity(quantity: number): Money;     // custo de N itens da faixa
    dividedByCount(count: number): Money;             // CPA: ads / nº pedidos (count 0 → zero)
    percentageOf(total: Money): number;               // margem %: this/total*100
    toCents(): number;
    isPositive(): boolean;
}
```

> **Sem `MarginHealth`.** Não há conceito de "saúde da margem". A margem é apenas **R$ e %**.
>
> **`Barcode`** — string validada (prefixo `T` + nanoid). **`Period`** — `{ start: Date; end: Date }` resolvido em SP.

**Modelo de erros (§9):**

Resposta de erro (única forma):

```jsonc
{
    "error": {
        "code": "TIER_COST_MUST_BE_POSITIVE",     // UPPER_SNAKE_CASE, nosso
        "message": "Mensagem técnica em inglês (log/dev)",
        "fields": { "costReais": ["Informe um custo válido"] }   // só em VALIDATION_ERROR
    }
}
```

> | Código | HTTP | Origem |
> | --- | --- | --- |
> | `VALIDATION_ERROR` | 400 | rota (Zod) |

> | `CATEGORY_NOT_FOUND` | 404 | repo |
> | `TIER_NOT_FOUND` | 404 | repo |
> | `ORDER_NOT_FOUND` | 404 | repo |

Mapeamento central na rota — helper `toHttpResponse(error)` em `src/lib/http-error.ts`:

```ts
export class DomainError extends Error {
    constructor(readonly code: string, readonly httpStatus: number, message: string) { super(message); }
}
export class TierCostMustBePositiveError extends DomainError {
    constructor() { super("TIER_COST_MUST_BE_POSITIVE", 422, "Tier cost must be positive"); }
}
```
