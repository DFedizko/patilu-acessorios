## 1. Objetivo (o que este RF entrega)

Permitir que a loja **leia e ajuste o custo fixo por pedido** (ex.: embalagem, hoje R$3), valor único compartilhado usado nos cálculos de lucro do Histórico e do Dashboard. Tecnicamente: a porta `IConfigPersistenceGateway`, sua implementação Prisma sobre o singleton `AppConfig`, os casos de uso `GetFixedCostUseCase`/`SetFixedCostUseCase` e as rotas `GET`/`PUT /api/config/fixed-cost`.

## 2. Contexto (por quê)

O lucro real do período (Histórico/Dashboard) desconta um **custo fixo por pedido** (embalagem etc.). Esse parâmetro precisa ser **único para a loja** (compartilhado entre operadores) e editável. Este RF entrega o lado de back-end — leitura e escrita do valor — que o RF-11 (front) expõe na aba "Custos fixos" do modal de Configurações e que o RF-16 consome no cálculo de margem líquida e lucro. É o **único RF de back-end deste lote** (RF-09 e RF-11 são front).

## 3. Início — pré-condições (o que já precisa existir)

- **RF-01 `DONE`:** `prisma/schema.prisma` já contém o model `AppConfig` (singleton); migração aplicada (tabela `app_config` existe); VO `Money`; `DomainError` + `toHttpResponse`; container DI back + `symbols`; banco de teste + helpers.
- **RF-02 `DONE`:** `HttpClient` back e front, `ApiError`, dicionário de erros PT, `QueryClient` `onError`.
- **RF-03 `DONE`:** `src/lib/schemas.ts` (com convenção de DTO `z.input`), `src/lib/openapi.ts`, rotas `/api/openapi.json` e `/api/reference` no ar.
- Como conferir: `bun run db:migrate` aplicou a tabela `app_config`; `/api/reference` abre o documento (ainda sem este endpoint).
- **Não altere `prisma/schema.prisma`** (criado só pelo RF-01 — o model `AppConfig` já está lá).

## 4. Escopo — entregáveis (faça exatamente isto)

- `src/server/application/gateway/IConfigPersistenceGateway.ts` — porta (PersistenceGateway, camada de aplicação): `getFixedCostPerOrderCents(): Promise<number>` e `setFixedCostPerOrderCents(cents: number): Promise<void>` (ler/gravar o singleton). **É PersistenceGateway** (não Repository): config simples que não mapeia um agregado de domínio.
- `src/server/infrastructure/gateway/ConfigPrismaPersistenceGateway.ts` — implementação Prisma sobre `AppConfig` (id `"singleton"`). Faz **upsert** do singleton ao gravar; ao ler, retorna o `fixedCostPerOrderCents` (com o default `300` do schema quando ainda não existir linha).
- `src/server/application/use-case/GetFixedCostUseCase.ts` + `.../contracts/IGetFixedCostUseCase.ts` — sem entrada → `{ fixedCostPerOrderCents: number }`. `Input` é vazio/`void`.
- `src/server/application/use-case/SetFixedCostUseCase.ts` + `.../contracts/ISetFixedCostUseCase.ts` — `{ amountReais }` → `{ fixedCostPerOrderCents: number }`. Converte reais→centavos via `Money.fromReais(...)` e grava. `Input = SetFixedCostDTO`.
- `src/lib/schemas.ts` — `setFixedCostSchema` (`amountReais` ≥ 0) + `export type SetFixedCostDTO = z.input<typeof setFixedCostSchema>;` (convenção §6.1).
- `src/app/api/config/fixed-cost/route.ts` — handlers `GET` (lê) e `PUT` (valida com `setFixedCostSchema`, grava). Auth (Clerk) + `try/catch` chamando `toHttpResponse`. Resposta em **centavos** (`{ fixedCostPerOrderCents }`).
- `src/lib/openapi.ts` — registrar `GET` e `PUT /api/config/fixed-cost` (entrada `{ amountReais }`, resposta `{ fixedCostPerOrderCents }`) + erro `VALIDATION_ERROR`.
- `src/lib/error-messages.ts` — garantir entrada PT para `VALIDATION_ERROR` (se ainda não existir).
- `src/server/di/container.ts` + `symbols.ts` — bindings de `IConfigPersistenceGateway`/`ConfigPrismaPersistenceGateway` + `GetFixedCostUseCase` + `SetFixedCostUseCase` (edição **aditiva** dos hotspots).

### Fora do escopo (NÃO faça)

- Não consumir o custo fixo em cálculo de lucro/margem — isso é do RF-16 (`PeriodReportCalculator`) e RF-17/RF-18.
- Não criar o **front** do modal/aba "Custos fixos" — RF-11.
- Não criar outras configs (conexão TikTok, impressora) — fora desta fase (PRD RN-7.3).
- Não congelar o custo fixo por pedido no empacotamento — decisão TS17 é **valor atual aplicado ao período**.
- Não alterar `prisma/schema.prisma`.

## 5. Fim — Definition of Done

- `GET /api/config/fixed-cost` devolve `{ fixedCostPerOrderCents }` (default `300` quando nunca configurado).
- `PUT /api/config/fixed-cost` com `{ amountReais }` ≥ 0 grava o singleton e devolve o novo valor em centavos; `amountReais` inválido (negativo/não numérico) → `VALIDATION_ERROR` (400).
- O valor é **único e compartilhado** (singleton `AppConfig`, id `"singleton"`).
- `SetFixedCostUseCase`/`GetFixedCostUseCase` dependem da **interface** `IConfigPersistenceGateway` (DIP), nunca do Prisma direto; dinheiro sempre via `Money`/centavos.
- Rotas protegidas por Clerk; erros mapeados por `toHttpResponse`.
- OpenAPI atualizado (caminhos GET/PUT + erro).
- **Testes da §14.2 passando** (`bun test`): integração `Get/SetFixedCostUseCase`.
- `bun run lint` limpo; sem `any`.

## 6. Testes que cobrem este RF (§14.2)
