## 1. Objetivo (o que este RF entrega)

A loja vê a **fila de pedidos do TikTok** por período, com os **pendentes no topo**, e abre o empacotamento de um pedido com um clique. Tecnicamente: renomear/reestruturar a página "Empacotar" para **`src/app/pedidos/page.tsx`** (Server Component), **componentizar a tabela** do Histórico em `components/ui` e reusá-la, reaproveitar o seletor de período do Dashboard, ligar a lista à API real (`GET /api/orders`) via service → hook (query) e navegar para a rota dinâmica `/pedidos/[orderId]/empacotar`.

## 2. Contexto (por quê)

Hoje a página "Empacotar" cria a caixa "do zero" com cliente/venda/frete digitados, a partir de **seed em memória** (Zustand). Neste PRD os pedidos passam a vir do **TikTok** (D14/D15): a página vira **"Pedidos"**, uma lista por período com os pendentes no topo em ordem de chegada (RN-2.2), e o botão "Empacotar" abre a tela do pedido pelo **id na URL** (RN-2.4). A tabela do Histórico e a de Pedidos devem **compartilhar o mesmo componente** (PRD §6 — reúso) — se ainda não estiver componentizada, componentizar agora e usar nos dois lugares (RN-2.3-tabela / D16).

## 8. Trecho da TECH_SPEC que fala deste RF

Telas afetadas (§13.4):

> - **Renomear "Empacotar" → "Pedidos"** (`src/app/pedidos/`): lista por período (reusa seletor do Dashboard); pendentes no topo; botão "Empacotar" → navega para `/pedidos/[orderId]/empacotar` (rota dinâmica). **Componentizar a tabela** hoje no Histórico em `components/ui` e reusar.
> - **Sem stores de seed:** trocar Zustand-seed por hooks de query/mutation reais (o catálogo/pedidos/ads passam a vir da API). Zustand fica só para estado de UI genuinamente compartilhado (ex.: rascunho do empacotamento, modal de etiqueta).

Estrutura por camada (§13.1):

> `component → hook (query|mutation) → service → HttpClient → API`. DI funcional via Inversify (`src/di/container.ts` + `symbols.ts`): services (factory functions) recebem o **`HttpClient`** (factory `axiosHttpClient`, §7.4) por injeção; hooks pedem a interface do service. Resolver pelo container dentro da factory. Nenhum service chama `axios`/`fetch` direto — sempre via `HttpClient`.

> **Localização das camadas no front:** a camada de **service fica em `src/service/`** (diretamente sob `src/`, **não** dentro de `src/hooks/`). `src/hooks/query/` e `src/hooks/mutation/` ficam separados e dependem da interface do service.

DTO de leitura consumido (§8):

type OrderListItem = {
