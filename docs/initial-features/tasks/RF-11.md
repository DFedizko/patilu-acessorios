## 1. Objetivo (o que este RF entrega)

Dar à loja um **modal de Configurações** acessível no rodapé da barra lateral, com a aba **"Custos fixos"** (editar o custo fixo por pedido) e a aba **"Perfil"** (nome/email/foto do Clerk + sair). Tecnicamente: um **componente `Modal` reutilizável** em `components/ui` (composition pattern + 15+ animações selecionáveis por prop `animation`/`direction`), e a ligação da aba de custo fixo à API real (`GET`/`PUT /api/config/fixed-cost`) via **service → mutation/query hook**.

## 2. Contexto (por quê)

O custo fixo por pedido entra no lucro de Histórico/Dashboard e precisa ser editável (PRD F7/D19). O lugar dele é um **modal de Configurações** no rodapé da sidebar, junto do perfil. Como o produto vai precisar de modais em vários lugares (etiqueta no RF-09, futuras abas no RF-21), este RF entrega o **primitivo `Modal`** reutilizável de uma vez — com a biblioteca de animações da TS14 — e o usa para construir o modal de Configurações. É **front**: liga a aba de custo fixo à API do RF-10 via service/hook; não há testes automatizados.

## 3. Início — pré-condições (o que já precisa existir)

- **RF-02 `DONE`:** `HttpClient` front (`createHttpClient` + `axiosHttpClient`), `ApiError`, dicionário `ERROR_CODE→PT`, `QueryClient` com `onError` global (toast sonner); container DI front (`src/di/container.ts` + `symbols.ts`) ligando o símbolo `HttpClient`.

- **Pages permanecem Server Components** — extrair ilhas client; **não** transformar a página inteira em `"use client"`.

## 4. Escopo — entregáveis (faça exatamente isto)

**Componente `Modal` (primitivo reutilizável em `src/components/ui/modal/`):**

- `Modal.tsx` — container client: overlay, foco, fechar no `ESC`, animação por prop. Props: `{ open, onOpenChange, animation?, direction?, children }`. **Composition pattern** (recebe `ModalHeader`/`ModalContent`/`ModalFooter` como `children`).
- `ModalHeader.tsx` — título + botão de fechar.
- `ModalContent.tsx` — corpo (`children: ReactNode`).
- `ModalFooter.tsx` — ações.
- `animations.module.css` — **15+ keyframes** (CSS modules) cobrindo `fade`, `zoom`, `slide`, `slide-corner`, `flip-x`, `flip-y`, `rotate-in`, `bounce`, `swing`, `pop`, `blur-in`, `elastic`, `fold`, `glitch`, `drop`, `reveal` (+ a opção `"random"` na prop). As **8 direções** (`top-left`, `top`, `top-right`, `right`, `bottom-right`, `bottom`, `bottom-left`, `left`) geram as variantes direcionais (ex.: `slideTopRight`).
- `use-modal-animation.ts` — resolve a classe final combinando `animation` + (quando direcional) `direction`; suporta `"random"` (sorteio variado por contador/efeito, **não** `Math.random` em SSR; animação centrada ignora `direction`).
- Padronizar pelos primitivos existentes (`Button`, `Field`); **relativo apenas** (sem `px` nas classes — usar tokens/utilitários Tailwind v4 ou `rem` por último).

**Modal de Configurações (consome o `Modal`):**

- Disparado por um botão no **rodapé** de `src/components/layout/sidebar.tsx` (estado de abertura **local** com `useState` na ilha client; Zustand só se precisar abrir de lugares distantes). Manter a sidebar como Client Component que já é, extraindo o gatilho/modal numa ilha própria se preferir.
- Modal com **abas**: **"Custos fixos"** e **"Perfil"**.
- Aba **"Custos fixos"**: formulário (React Hook Form + Zod, reusando `setFixedCostSchema`/`SetFixedCostDTO` de `@/lib/schemas` quando o payload casar) com o campo **custo fixo por pedido** (usar `MoneyInput`); carrega o valor atual (query) e salva (mutation) contra `GET`/`PUT /api/config/fixed-cost`.
- Aba **"Perfil"**: `useUser()` (nome, email, foto) + botão **Sair** com `useClerk().signOut()`. UI custom com nossos primitivos (Tailwind), **sem** componentes prontos do Clerk.

**Camadas front (component → hook → service → HttpClient):**

- `src/service/config-service.ts` — factory que recebe o `HttpClient` (interface) e expõe `getFixedCost()` (`GET /api/config/fixed-cost`) e `setFixedCost(input: SetFixedCostDTO)` (`PUT /api/config/fixed-cost`).
- `src/hooks/query/use-fixed-cost.ts` — `useQuery` (queryKey consistente, ex.: `["config", "fixed-cost"]`) pedindo a interface do service via container.
- `src/hooks/mutation/use-set-fixed-cost.ts` — `useMutation` que, no sucesso, **invalida** a queryKey `["config", "fixed-cost"]`.
- `src/di/container.ts` + `symbols.ts` — registrar o `config-service` e ligá-lo ao `HttpClient` (edição **aditiva** dos hotspots front).

## 8. Trecho da TECH_SPEC que fala deste RF

Decisão técnica (§1):

> | TS14 | **Front-end:** `Modal` reutilizável em `components/ui` com composition pattern + **15+ animações** (CSS modules) selecionáveis por prop `animation` (com `"random"`) e prop `direction` (8 origens; ignorada por animações centradas). |

Componente `Modal` (§13.3):

> `src/components/ui/modal/` :
>
> ```
> modal/
> ├── Modal.tsx            # container (overlay + foco + ESC + animação); props: { open, onOpenChange, animation, direction, children }
> ├── ModalHeader.tsx      # título/!-fechar
> ├── ModalContent.tsx     # corpo (children: ReactNode)
> ├── ModalFooter.tsx      # ações
> ├── animations.module.css  # 15+ keyframes (CSS modules)
> └── use-modal-animation.ts # resolve a classe da animação a partir de (animation, direction); suporta "random"
> ```

> ```tsx
> type ModalAnimation =
>     | "fade" | "zoom" | "slide" | "slide-corner" | "flip-x" | "flip-y"
>     | "rotate-in" | "bounce" | "swing" | "pop" | "blur-in" | "elastic"
>     | "fold" | "glitch" | "drop" | "reveal" | "random";
>
> // 8 direções de origem da animação (de onde o modal entra)
> type ModalDirection =
>     | "top-left" | "top" | "top-right" | "right"
>     | "bottom-right" | "bottom" | "bottom-left" | "left";
>
> type ModalProps = {
>     open: boolean;
>     onOpenChange: (open: boolean) => void;
>     animation?: ModalAnimation;   // default: "fade"
>     direction?: ModalDirection;   // default: "bottom" (só aplicada a animações direcionais)
>     children: React.ReactNode;
> };
>
> // uso (composition):
> <Modal open={open} onOpenChange={setOpen} animation="slide" direction="top-right">
>     <ModalHeader>Configurações</ModalHeader>
>     <ModalContent>{/* abas: Custos fixos, Perfil */}</ModalContent>
>     <ModalFooter>{/* ações */}</ModalFooter>
> </Modal>
> ```

> - **`direction` (8 valores):** define **de onde** a animação entra — `top-left`, `top`, `top-right`, `right`, `bottom-right`, `bottom`, `bottom-left`, `left`.
> - **Animações direcionais vs centradas:** animações que têm origem espacial (ex.: `slide`, `slide-corner`, `fold`, `drop`, `reveal`) **usam** a `direction` para escolher o keyframe/transform de entrada. Animações que vêm do **centro da tela** (ex.: `fade`, `zoom`, `flip-x`, `flip-y`, `rotate-in`, `pop`, `blur-in`, `glitch`) **ignoram** a `direction`.
> - `use-modal-animation.ts` resolve a classe final combinando `animation` + (quando direcional) `direction` — ex.: `slide` + `top-right` → classe `slideTopRight` do `animations.module.css`. Para `"random"`, sorteia uma animação e, se ela for direcional e nenhuma `direction` foi passada, sorteia também a direção (sorteio variado por contador/efeito, **não** `Math.random` em SSR).
> - Estado de abertura: **local** (`useState`) no consumidor (ex.: botão do rodapé da sidebar). Zustand só se precisar abrir o mesmo modal de lugares distantes.

Autenticação (Clerk) — UI custom (§12):

> - **Ler usuário:** servidor `currentUser()` (`fullName`, `primaryEmailAddress.emailAddress`, `imageUrl`); client `useUser()`.
> - **UI custom (não usar componentes prontos):** sign-in com `useSignIn().sso({ strategy: "oauth_google", ... })` + página `/sso-callback`. **Modal de Configurações** lê `useUser()` (nome/email/foto) e sai com `useClerk().signOut()`. Estilizar com nossos primitivos (Tailwind), sobrepondo o `appearance` quando algum componente do Clerk for usado.
> - Erros de auth → toast amigável PT (nunca texto cru do Clerk).

Telas afetadas (§13.4):

> - **Modal de Configurações** no rodapé da sidebar: aba **"Custos fixos"** (custo fixo por pedido) + **Perfil** (nome/email/foto do Clerk, sair).

Estrutura por camada e erros (§13.1/§13.2):

> `component → hook (query|mutation) → service → HttpClient → API`. DI funcional via Inversify (`src/di/container.ts` + `symbols.ts`): services (factory functions) recebem o **`HttpClient`** por injeção; hooks pedem a interface do service.

> - `QueryClient` com `onError` global (queries e mutations) → `resolveMessage(error)` → `toast.error(...)` (sonner). Hooks específicos podem sobrescrever quando precisarem de UX diferente.

Linha do catálogo (§18.2 — a espinha):

> | **RF-11 (front)** | Componente `Modal` reutilizável (composition + 15+ animações + `animation`/`direction`, §13.3) + **modal de Configurações** (aba Custos fixos + Perfil do Clerk). | RF-02, RF-04, RF-10 | endpoint de custo fixo + Clerk | modal abre da sidebar; salva custo fixo; mostra perfil. **Testes:** não (front). |
